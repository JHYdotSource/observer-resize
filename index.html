Resize Event Listener vs. Resize Observer vs. Container Queries

Vorteile/Nachteile

  • Nur für das window Element verfügbar
  • Kein direkter bezug zum Element das sich verändert
  • Das Element kann sich verändern ohne den EventListener zu triggern
  • Wird nicht inital getriggert (zusätzliches JS notwendig)
  • Klassen/Attribute hinzufügen/entfernen
  • Markup verändern
  • Auch für alte Browser verfügbar
  • Nicht einfach wiederzuverwenden
  • Named Function in EventListener sonst kein Entfernen möglich

  • Wird initial getriggert auch ohne Resize
  • Browser Support seit 2020
  • Klassen/Attribute hinzufügen/entfernen
  • Markup verändern
  • Einfache wiederverwendung für andere Elemente
  • Inline Size und Block Size

Container Queries

Vorteile/Nachteile

  • Kein JS notwendig
  • Wird nur von modernen Browsern unterstützt
  • Kein Zugriff auf Parent
    • Zugriff auf Parent via :has() Selector
  • Markup bleibt gleich
  • Klassen/Attribute bleiben gleich
  • Einfaches verschachteln von Media-Queries und Container-Queries
  • Einfache wiederverwendung bei beliebig vielen Elementen
  • Inline Size und Block Size oder nur Inline Size
